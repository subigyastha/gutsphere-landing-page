3. Phase 1 — Create Pages project Cloudflare UI

Connect the GitHub repo and configure the first production build.

  1. Open Cloudflare Dashboard Go to Workers & Pages → Create → Pages → Connect to Git.
  2. Authorize GitHub Select the Gutsphere Landing repository and grant access if prompted.
  3. Enter build settings exactly
    FieldValue
    Project namee.g. gutsphere-landing
    Production branchmain
    Framework presetVite (or None)
    Build commandnpm run build
    Build output directorydist
    Root directory/ (leave default)
    Env varNODE_VERSION = 20

7. Phase 5 — DNS cutover from Webflow Production

One-time change in Cloudflare DNS. App / demo / newsletter subdomains stay untouched.

  1. Day before: lower TTL On the www record, set TTL to ~300 seconds if it is currently high.
  2. Confirm Pages deploy is green Latest production deployment on Cloudflare Pages must be healthy.
  3. Add custom domain in Pages Pages project → Custom domains → add www.gutsphere.com. Cloudflare usually creates/updates the DNS record in the same zone.
  4. Point www at Pages Remove or replace the old Webflow CNAME. Use proxied (orange cloud) if that is your normal setup.
  5. Apex redirect Redirect Rule: gutsphere.com → https://www.gutsphere.com (301).
  6. SSL SSL/TLS mode: Full (strict).
  7. Do not change Leave alone: app, demo, newsletter DNS records.
  8. Keep Webflow as rollback Do not delete the Webflow project for 1–2 weeks.
  9. If assets look stale Caching → Purge Everything.
